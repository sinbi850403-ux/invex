INVEX는 중소기업 재고·입출고 핵심 기능에서 타사와 대등한 수준을 갖추고 있으며, 한국어 특화 세무·급여 모듈과 다크모드 등에서 명확한 우위를 보입니다. 그러나 실시간 데이터 갱신, 시리얼/LOT 추적, 커스텀 리포트 등 중급 사용자가 필요로 하는 기능에서 공백이 존재해 업셀링과 이탈 방어에 리스크가 됩니다.

5. INVEX 강점 및 약점

강점 (Strengths)

  • 한국어 특화 — 세무·급여·연말정산 내장
  • 다크모드 기본 지원 (타사 모두 미지원)
  • 위젯 드래그 커스터마이징
  • 역할별 대시보드 (경영자/직원)
  • AI 수요 예측 탑재
  • 주간 보고서 자동화
  • Free 플랜 제공 (100품목 영구 무료)
  • 1년 전체 기능 무료 개방 정책
  • 빠른 로딩 속도 (Vite + Supabase)

약점 (Weaknesses)

  • 실시간 자동 갱신 미지원 (새로고침 필요)
  • 시리얼/LOT 번호 추적 없음
  • 로케이션(선반/구역) 관리 없음
  • 재고 조정 이력 미구현
  • 비교 기간 분석 없음
  • 커스텀 리포트 빌더 없음
  • 목표치/달성률 KPI 없음
  • 모바일 UX 완성도 낮음

6. 개선 우선순위 로드맵

HIGH

실시간 데이터 갱신

Supabase Realtime 구독으로 새로고침 없이 KPI 자동 업데이트. 협업 환경에서 데이터 불일치 해소. 구현 난이도 낮음, 임팩트 높음.

HIGH

시리얼/LOT 번호 추적

제조업·식품·의료 고객 온보딩의 핵심 블로커. 해당 산업 고객의 이탈을 막으려면 우선 구현 필요. DB 컬럼 추가 + UI 입력 필드.

MEDIUM

비교 기간 분석 (이번 달 vs 지난 달)

경영자가 가장 자주 묻는 "지난달 대비 얼마나 좋아졌나"에 답하는 뷰. 고급분석 페이지에 기간 비교 토글 추가로 구현 가능.

MEDIUM

재고 조정 이력 추적

감사·회계 목적으로 재고 수량 변경 이력을 별도 테이블에 기록. 이카운트 전환 고객이 필수로 요구하는 기능.

LOW

로케이션(선반/구역) 관리

창고 내 세부 위치 지정. 물류 창고 고객 확장 시 필요. 현재 다창고 구조를 기반으로 하위 레벨 추가 형태로 설계 가능.

LOW

목표치 설정 및 달성률 표시

월 목표 매출·입고량을 설정하고 KPI 카드에 진행률 바 표시. user_settings 테이블 활용으로 추가 DB 불필요.

7. 단계별 개발 로드맵

1

즉시 (1~2개월) — 이탈 방지

현재 사용자 이탈을 막는 기능 보완. 구현 난이도 낮고 효과 즉시.

실시간 갱신 (Supabase Realtime) 재고 조정 이력 비교 기간 분석 모바일 UX 개선
2

단기 (3~5개월) — 신규 고객 확장

제조·식품·물류 업종 고객 온보딩을 가능하게 하는 기능 추가.

시리얼/LOT 추적 로케이션 관리 목표치/달성률 KPI 커스텀 필드 확장
3

중기 (6~12개월) — Enterprise 업셀링

대형 고객 전환 및 Enterprise 플랜 가치 증대.

커스텀 리포트 빌더 API Webhook 강화 모바일 앱 (PWA 고도화) 멀티 테넌시
